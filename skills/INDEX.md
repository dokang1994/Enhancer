# Skill Catalog

Status: Proposed

This index contains candidate Skills accepted for future design work. Entries remain unavailable until a validated `skills/<name>/SKILL.md` exists and the entry status changes to `Available`. See RFC-0007 and `.ai/skill_rules.md`.

| name | status | description (when to use) | allowed-tools |
|---|---|---|---|
| brainstorming | Proposed | Use when intent and design need exploration before creative implementation. | read |
| test-driven-development | Proposed | Use when observable feature or bug-fix behavior is implemented. | read, execute, write-code |
| systematic-debugging | Proposed | Use when a bug, failing test, or unexpected behavior requires diagnosis. | read, execute |
| verification-before-completion | Proposed | Use immediately before claiming completion, passing checks, or a fix. | read, execute |
| receiving-code-review | Proposed | Use when technical review feedback must be evaluated. | read |
| writing-plans | Proposed | Use when a multi-step implementation plan is required. | read, write-docs |
| writing-skills | Proposed | Use when a repository Skill is created, changed, or validated. | read, execute, write-docs |
