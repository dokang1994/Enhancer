# Coding Rules

- Use Java 17.
- Prefer Spring Boot 3 when application wiring is needed.
- Use Gradle as the build system.
- Use JUnit5 and Mockito for tests.
- Keep code minimal.
- Follow SOLID.
- Avoid DDD in the early phase.
- Avoid unnecessary abstractions.
- For observable feature and bug-fix behavior, write a failing test first, confirm the expected failure, and then implement the minimum change. Documentation-only and behavior-preserving mechanical changes may use an appropriate alternative verification.
