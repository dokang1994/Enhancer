# AI Workflow

Enhancer follows Document Driven Development.

Before every session:

```text
항상 .ai 폴더를 읽고 시작해.
```

Then:

1. Read `CONSTITUTION.md` and confirm the work fits the project rules.
2. Read or update `ARCHITECTURE.md`.
3. Record accepted decisions in `DECISION_LOG.md`.
4. Define or confirm the current task in `CURRENT_TASK.md`.
5. Implement the smallest scoped change.
6. Run relevant compile and test checks.
7. Update project documents.
8. Update `SESSION_HANDOFF.md`.
9. Commit only when required.
