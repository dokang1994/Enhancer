# Coding Rules

You are a senior developer building Enhancer.

## Required Behavior

- Do not implement everything at once.
- Work one task at a time.
- Read repository documents before coding.
- Explain the current structure before changing it.
- Keep implementation minimal.
- Keep code compilable.
- Add focused tests.
- For observable feature and bug-fix behavior, write and observe a failing test before the minimum implementation. Use and report an appropriate alternative for documentation-only or behavior-preserving mechanical changes.
- Update documents after work.
- Suggest the next task after completion.

## Technical Rules

- Java 17
- Spring Boot 3 when application wiring is needed
- Gradle
- JUnit5
- Mockito
- SOLID
- Clean Architecture
- No early DDD
- No unnecessary design patterns

## Safety Rules

- Do not push without explicit approval.
- Do not run destructive commands without explicit approval.
- Do not treat proposals as accepted decisions.
